# Runbook — the human layer (Finding 2G)

Scoped narrowly on purpose. An over-scoped runbook invites a well-meaning second
person to improvise past their competence — that's a new risk, not a removed one.

## Backup policy DECISION (resolving 2G — the actual hole)

"Generated on-node" is narrower than "the key exists in exactly one place."
Disk snapshots silently duplicate the private key into provider-custody storage
with no event marking the moment the guarantee died.

**DECISION: Option (a) — node private keys are EXCLUDED from all disk backups/
snapshots.** The key directory `/etc/nexinus/` must never be in any snapshot
scope. A destroyed node means full re-keying as part of recovery — which is cheap
because rotation is already "generate on-node, publish the new public key." We do
NOT accept provider-custody key copies. If this is ever reversed, the guarantee
is explicitly downgraded to "one live place + provider backups" and Hetzner
account compromise becomes a key-compromise vector — must be re-documented here.

## THE ONE RULE — as a provisioning checkbox (not a principle to remember)

Run this check at every provision, out loud:

> [ ] Was this private key generated by running keygen ON THIS BOX, in THIS
>     session, right now? If it was copied, SCP'd, or pasted from anywhere —
>     STOP, discard it, regenerate on-node.
> [ ] Did I copy ONLY the .pub (public) file off the box? Never node.key.
> [ ] Is /etc/nexinus excluded from snapshots/backups?

Fatigue note: never `cat node.key` "just to check it's not empty" — that's the 2am
mistake. Check `node.pub` only. Disable cloud-sync clipboard managers during
key work.

## Recovery checklists (ordered, 2am-followable — not prose)

### Node down past TTL
1. Status page shows it UNKNOWN/DOWN — that is correct, not a bug.
2. Open Hetzner console → is the box Running? If off, power on.
3. SSH in (or web console). Check: `systemctl status caddy nexinus-sign.timer`.
4. If timer dead: `systemctl restart nexinus-sign.timer`.
5. Verify from laptop: `curl https://<domain>/health` returns signed JSON.

### GitHub account locked
1. Use offline recovery codes (see map for location).
2. Repo also lives at the bootstrap mirror (see map) — deploy from there.

### Domain near-lapse
1. Namecheap → enable auto-renew + backup card. Confirm not expired.
2. If account compromised: re-point via exported zone file at backup provider.

### Valkyrie destroyed
1. Re-provision box. Run its setup script (regenerates its OWN keypair).
2. Commit the NEW public key to `pubkeys/`. Old key is dead — that's fine.
3. Aggregator history is gone → treat as forced re-baseline (see VERIFIER 2E).

## Second-person boundary (what they must NOT do)

ALLOWED: restart a node, read the status page, understand STALE/DOWN/UNKNOWN,
execute a documented smallest-reversible action above.
OUT OF SCOPE for anyone but the operator: key rotation, signing-scheme changes,
anything not written as a checklist here. When in doubt: do nothing, wait.
